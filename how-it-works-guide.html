NC AI Office Agent — How It Works

A plain-English guide to what the system does, how data flows, and why it's built this way.

The 30-Second Pitch

The Governor's comms staff open a Microsoft Copilot Studio chatbot in Teams and ask questions in plain English. Behind the scenes, Azure Functions call Azure OpenAI (GPT-4o) and Azure AI Search to find the right information, summarize it, and send back an answer — all in a few seconds.

The system does three things:

1. News Clips — Automatically collects press releases mentioning the Governor, so staff can ask "What's in the news today?"

2. Remarks Search — Lets staff search the Governor's past speeches by topic, so they can ask "What has the Governor said about education?"

3. Transcript Proofreading — Cleans up rough transcripts (from voice-to-text or scanned docs), fixing typos and formatting.

What Staff Actually See

Staff never see Cosmos DB, Azure Functions, or APIM. They see a Copilot Studio chatbot in Microsoft Teams (or embedded on a web page). It works like texting a very knowledgeable assistant. Entra ID SSO handles auth — if they can open Teams, they're already signed in.

Capability 1: News Clips

Automatically collects Governor press releases, then lets staff search or browse them instantly.
How data gets in (runs automatically every 15 minutes):
1 Fetch An Azure Function (a small piece of serverless code) wakes up on a 15-minute timer. It scrapes governor.nc.gov/news/press-releases and reads new articles using Mozilla Readability to extract clean text.
→
2 Read & Understand Azure OpenAI (text-embedding-3-large model) converts each article's title + lede into a vector embedding — a long number sequence that captures its meaning, not just keywords.
→
3 Store The article + its embedding are saved in two places: Cosmos DB (the NoSQL database — for browsing "latest clips") and Azure AI Search (the search index — for hybrid keyword + semantic search).
Analogy: Imagine a news intern who checks the Governor's website every 15 minutes, reads every new article, summarizes it on a card, and files it in both a filing cabinet (for "show me everything") and a magic Rolodex (for "find me articles about X").
How staff ask questions:
1 Staff Asks Staff asks in Copilot Studio (Teams chatbot): "What clips came in today?" or "Find clips about broadband"
→
2 Hybrid Search Azure OpenAI converts the question into an embedding. Azure AI Search runs a hybrid query — matching by both vector similarity (meaning) and BM25 (keywords) — then re-ranks with its semantic ranker.
→
3 Results Top matching clips flow back through APIM → Copilot Studio → the user's Teams window, with title, date, source, and the key quote.
Key insight: Searching for "internet access in rural areas" will find an article titled "Broadband Investment for Underserved Communities" even though they share zero words — because the meaning matches. This is what makes it smarter than a simple keyword search.
🎤

Capability 2: Remarks Search

Upload the Governor's speeches and search them by topic. The AI reads all matching passages and writes a summary with direct quotes.
How speeches get in (happens when a file is uploaded):
1 Upload A staff member drops a speech file (.txt) into a Blob Storage container (remarks-uploads). This automatically triggers an Azure Function (blob trigger).
→
2 Chop Into Pieces The Azure Function splits the speech into paragraph-sized chunks (~200–1,000 chars each) so the search can pinpoint exactly where a topic is discussed. Metadata (date, event, venue) is parsed from the filename.
→
3 Understand & Index Azure OpenAI (text-embedding-3-large) generates an embedding for each chunk. Chunks are stored in Cosmos DB and indexed in Azure AI Search, tagged with date, event, and venue.
Analogy: Imagine highlighting every paragraph of every speech the Governor has ever given, writing a one-line summary of what it's about on a sticky note, and putting every sticky note in a searchable binder sorted by topic. That's what the system builds automatically.
How staff ask questions:
1 Staff Asks Staff asks in Copilot Studio: "What has the Governor said about teacher pay?" — routed through APIM to the remarks Azure Function.
→
2 Hybrid Search Azure OpenAI embeds the question. Azure AI Search runs a hybrid query (vector + BM25 + semantic reranking) and returns the top 10 most relevant speech chunks across all speeches.
→
3 GPT-4o Synthesizes The Azure Function sends those chunks to Azure OpenAI (GPT-4o), which writes a synthesis with direct quotes, citations (date, event), and notes on how messaging evolved over time.
This is the RAG pattern (Retrieval-Augmented Generation): instead of asking AI to make something up, we retrieve real source material first, then ask AI to summarize only what it found. The AI never invents quotes — it cites real text from real speeches.
✍️

Capability 3: Transcript Proofreading

Paste a rough transcript, get back a clean version with every change explained.
1 Staff Pastes Text Staff pastes a raw transcript in Copilot Studio — from voice-to-text software or a scanned doc. Routed through APIM to the proofread Azure Function.
→
2 GPT-4o Proofreads The Azure Function sends the text to Azure OpenAI (GPT-4o) with a specialized proofreading prompt. GPT-4o fixes ASR/OCR errors, punctuation, proper nouns — but never changes meaning or speaking style.
→
3 Structured Output GPT-4o returns structured JSON: the corrected text, a list of every change with a reason, and a confidence level (high/medium/low). No data is stored — nothing touches Cosmos DB or Blob Storage.
Analogy: Like handing a rough draft to the best copy editor in the building — except they work instantly and show you every single red-pen mark with an explanation of why they changed it.

Where Does Data Live?

What Where It's Stored Why There
News articles (full text, dates, sources) Cosmos DB (database) Fast lookups, browsing "latest clips," flexible schema
News article meaning-codes (embeddings) Azure AI Search (search index) Powers "find clips about X" — semantic + keyword hybrid search
Speech chunks (paragraphs + metadata) Cosmos DB + Azure AI Search Database for storage, search index for smart retrieval
Original speech files (.txt, .docx, .pdf) Azure Blob Storage (file storage) Cheap, durable file storage — uploading here triggers processing
Transcripts (proofread) Nowhere — processed in real time Text goes in, clean text comes back. Nothing is saved.

How the Pieces Connect (The Plumbing)

You don't need to explain this to most audiences, but here's the short version if asked:

1. Microsoft Copilot Studio (the chatbot in Teams) receives the user's question. Its built-in intent recognition figures out which capability to call (clips, remarks, or proofread).

2. Copilot Studio sends the request through a Power Platform custom connector to Azure API Management (APIM) — the gateway. APIM validates the subscription key, enforces the 60 req/min rate limit, injects the Azure Function key, and routes to the correct endpoint.

3. Azure Functions (Flex Consumption, Node.js 20) — small TypeScript functions that wake up on demand — do the actual work. There are separate functions for clips-query, remarks-query, and proofread, plus timer/blob-triggered functions for ingestion.

4. The query functions call Azure OpenAI (East US 2) — specifically text-embedding-3-large to embed the question, and GPT-4o to synthesize answers or proofread text.

5. The query functions call Azure AI Search (Basic tier) for hybrid vector + keyword search with semantic reranking, and Cosmos DB (Serverless) for direct lookups and browsing.

6. The answer flows back: Azure Function → APIM → Custom Connector → Copilot Studio → the user's Teams chat window.

The entire round trip — question in, answer out — takes 3–8 seconds. All compute is serverless (Azure Functions Flex Consumption): when nobody is asking questions, Functions scale to zero and cost almost nothing. Cosmos DB is also serverless (pay-per-request). The only fixed-cost service is AI Search Basic at ~$70/mo.

Trust & Security (The Non-Negotiables)

🔐 No Passwords in Code Every service authenticates via Azure Managed Identity + DefaultAzureCredential. Functions connect to Cosmos DB, AI Search, OpenAI, and Key Vault without a single password or connection string in the code.
🏛️ Data Stays in Azure All data is processed within the state's Azure tenant. Nothing leaves to third-party services. Azure OpenAI runs inside the same cloud boundary.
🛡️ AI Content Filters Azure OpenAI has built-in responsible AI filters. The AI can only summarize what's in the real data — it's grounded, not generative.
👤 Entra ID SSO Staff log in with their existing state credentials. No new accounts. No extra passwords. If they can access Teams, they can access the agent.
🚦 Rate Limiting APIM (Consumption tier) enforces a 60 req/min rate limit per subscription key. Prevents runaway Azure OpenAI token costs and protects against abuse.
🔒 Private Networking Blob Storage (where speech files are uploaded) is locked behind a VNet private endpoint — no public internet access. Azure Functions reach it via VNet integration.

Talk Track — Common Questions

Q: Is the AI making things up?
No. For clips and remarks, the AI only summarizes real documents that are already in the system. It shows its sources with dates and direct quotes. This is called "grounded generation" — the AI is a research assistant, not a content creator.
Q: What happens if nobody uses it for a week?
Azure Functions scale to zero — they essentially sleep. Cosmos DB is serverless too (pay-per-request). You pay almost nothing when idle. The clips-ingest timer Function still fires every 15 minutes to check governor.nc.gov, but each run costs fractions of a penny. The only fixed cost is AI Search Basic (~$70/mo).
Q: How is this different from just Googling?
Three ways: (1) Azure AI Search uses hybrid search — vector embeddings (meaning) + BM25 (keywords) + a semantic reranker — so "rural internet access" finds "broadband investment." (2) GPT-4o reads all matching results and writes a synthesis with direct quotes. (3) For remarks, it searches the Governor's own speeches stored in our Cosmos DB and AI Search index, which aren't on Google.
Q: What if the AI gets something wrong?
Every answer includes its sources — staff can click through to the original article or speech chunk. For proofreading, every change is listed with a confidence level and a reason. Low-confidence corrections are flagged with [?].
Q: How much does this cost to run?
About $120–195/month for the whole platform — less than a single software license. Everything is serverless and consumption-based, so costs scale with actual usage.
Q: Can this be extended to other offices?
Yes. Each capability is a separate Azure Function with its own APIM route. You add a new capability by writing a new Function and adding a Copilot Studio topic. The entire infrastructure is defined in Bicep (infrastructure-as-code under /infra) — you can spin up a copy for another office with a parameter change and a 15-minute deploy.
Q: Why Teams and not a custom app?
Staff already live in Teams. No new app to install, no new password to remember, no training required. They just type a question like they're messaging a colleague. Copilot Studio handles the UI, auth, and deployment — zero frontend code needed.
