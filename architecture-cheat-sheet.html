NC DIT AI Office Agent — Architecture Decision Cheat Sheet

Why each Azure service was chosen, what it replaced, and why it's the right call for this workload.
Guiding Principles

Service-by-Service Justification

Cosmos DB (NoSQL) Serverless

Why Cosmos DB and not SQL or Table Storage?
  • Serverless billing — pay per request, not per hour. ~$5-20/mo for this workload vs. $24+/mo minimum for provisioned Cosmos or $5+/mo for SQL Basic
  • Schema flexibility — clips and remarks have different shapes; no rigid table schemas or migrations needed
  • Sub-10ms reads — instant response when Copilot Studio queries "today's clips"
  • Native RBAC — managed identity auth via sqlRoleAssignments; no connection strings
  • vs. SQL Server: overkill — no joins, no transactions, no relational needs
  • vs. Table Storage: no query flexibility, no indexing policy, no change feed

Azure AI Search Basic (B)

Why a separate search service?
  • Vector search — semantic similarity over embeddings; Cosmos DB can't do this
  • Hybrid search — combines vector (meaning) + BM25 (keywords) for best relevance
  • RAG pattern — purpose-built to feed top-K chunks to GPT-4o for grounded answers
  • Semantic ranker — ML re-ranking on top of hybrid results (available at Basic+)
  • vs. Cosmos full-text: no vector support, no hybrid ranking, no semantic reranker
  • vs. pgvector/PostgreSQL: always-on compute; AI Search Basic is fixed-cost, simpler ops

Azure Functions Flex Consumption

Why Functions, and why Flex specifically?
  • Scale to zero — no cost when nobody's asking questions; ~$5-15/mo
  • Native triggers — timer (clips ingestion), blob (remarks upload), HTTP (queries) all built-in
  • Flex Consumption — adds VNet integration (required for private endpoints) that classic Consumption lacks
  • TypeScript/Node 20 — fast cold starts (~200ms), same language across entire stack
  • vs. App Service: always-on minimum cost ($13+/mo B1), no timer/blob triggers
  • vs. Container Apps: more ops overhead, ~$5/mo minimum even scaled to zero

API Management (APIM) Consumption

Why put a gateway in front of Functions?
  • Auth boundary — Copilot Studio sends one subscription key; APIM injects function keys internally
  • Single endpoint — one base URL for all 3 capabilities (clips, remarks, proofread)
  • Rate limiting — 60 req/min cap protects against runaway costs on OpenAI/Search
  • Custom connector — Power Platform connector points to APIM, not raw Function URLs
  • vs. no gateway: would expose function keys to Copilot Studio; no rate limiting; harder to audit

Azure OpenAI Standard (East US 2)

Why Azure OpenAI vs. OpenAI directly?
  • Data residency — data stays in the Azure tenant; required for government workloads
  • Managed identity — Cognitive Services OpenAI User role; no API keys
  • Content filtering — built-in responsible AI filters (required for state government use)
  • GPT-4o at 30K TPM for synthesis/proofreading; text-embedding-3-large at 120K TPM for vectors
  • vs. OpenAI direct: API key auth, data leaves tenant, no Azure RBAC integration

Copilot Studio Per-tenant license

Why not build a custom web app?
  • Zero frontend code — ships in Teams, web, SharePoint embed instantly
  • Entra ID SSO — no auth plumbing; staff sign in with their existing credentials
  • Topic routing — built-in intent recognition maps user questions to the right backend action
  • Already licensed — most E3/E5 tenants include Copilot Studio per-tenant
  • vs. custom React app: months of frontend/auth work; ongoing maintenance; separate hosting cost

Key Comparison Points

A custom-built equivalent (App Service + SQL + React + auth) would cost $200-400/mo in compute alone, plus months of dev time on frontend, auth, and DevOps. This stack is fully IaC (Bicep) — teardown and redeploy in 15 minutes.
