NC Governor's Communications Agent · What it does, how it works, and how to demo it · March 2026

What This Is

A serverless AI platform built for the NC Governor's Communications Office. It does four things:

News Clips — "What's in the news about the Governor?" Every morning at 7 AM, the system scrapes governor.nc.gov press releases AND searches the web using Azure OpenAI with Bing grounding — pulling from WRAL, WUNC, AP, Charlotte Observer, and 20+ other outlets. Staff search clips in Teams using natural language. Currently: 78 clips from 29 outlets.
Remarks Search — "What has the Governor said about healthcare?" Upload a speech or column. The system chunks it into paragraphs, creates vector embeddings, and indexes it. Staff ask topic questions; GPT-4o synthesizes an answer with direct quotes and citations from across all speeches. Currently: 7 documents, 26+ chunks.
Transcript Proofreading — "Clean up this rough transcript" Paste a faulty transcript. GPT-4o fixes spelling, grammar, and punctuation. Every change is explained with a confidence level (high/medium/low) and a reason. Nothing is stored — text in, clean text out.
Audio/Video Transcription — "Transcribe this recording" Upload an audio or video file (up to 25MB). Azure OpenAI Whisper converts it to text. Returns the transcript with filename and file size metadata.

The Dashboard

A React SPA that visualizes the platform's data and health. Four tabs:

Overview
Stat cards (total clips, remarks, unique outlets, latest ingest), outlet breakdown bar chart, latest ingestion run status. Auto-refreshes every 60 seconds.
Clips
Paginated feed of all ingested clips. Filter by outlet (dropdown) and date range. Each card shows title, outlet, published date, lede, and links to the original article.
Remarks
Table of all uploaded speech/column documents with title, date, event, venue, chunk count, and ingestion timestamp.
Runs
History of every ingestion run — trigger type (timer/manual), duration, new/skipped/errors, source breakdown (gov + web), and status badge (success/partial/failed).

How News Clips Work

Ingestion (7 AM daily + manual refresh)

  1. Scrape — Azure Function fetches 2 pages of governor.nc.gov press releases (~20 articles)
  2. Web Search — 5 parallel Azure OpenAI Responses API calls with Bing grounding, each targeting a different topic (budget, Helene, healthcare, law enforcement, general). Returns ~30-40 unique external URLs.
  3. Merge & Dedup — Gov + web results merged, deduplicated by URL hash. Gov URLs filtered from web results (scraper already covers those).
  4. Extract — Each article fetched, parsed with Mozilla Readability for clean text extraction.
  5. Embed — Azure OpenAI text-embedding-3-large creates a vector embedding of each article.
  6. Store — Article + embedding saved to Cosmos DB (metadata) and Azure AI Search (hybrid search index).
  7. Log Run — Run results (counts, timing, status) persisted to Cosmos for dashboard display.

Query (staff asks in Teams)

  1. Question goes to Copilot Studio chatbot in Teams
  2. Chatbot calls APIM → Azure Function → AI Search (hybrid: vector + keyword + semantic reranker)
  3. Top matching clips returned with title, outlet, date, and Governor mention context
Key demo point: The daily timer uses "past week" for focused coverage. The manual refresh endpoint uses "past 6 months" for backfill. Cost: ~$0.175/day ($5/month) for the 5 web search calls.

Suggested Demo Flow

  1. Dashboard Overview — Open the dashboard. Show the stat cards (78 clips, 29 outlets). Point to the outlet breakdown. "This runs every morning at 7 AM automatically."
  2. Dashboard Clips — Switch to Clips tab. Filter by "WRAL" to show external media. "These are real articles from WRAL, pulled in via Azure OpenAI with Bing grounding — not scraping, not RSS."
  3. Dashboard Runs — Switch to Runs tab. Show the latest run — 24 new, 29 skipped, 0 errors, 47 seconds. "Every run logs itself so ops can see what happened."
  4. Teams Chatbot — Clips — Switch to Teams. Ask: "What clips came in about broadband?" Show the semantic search results with outlet and date.
  5. Teams Chatbot — Remarks — Ask: "What has the Governor said about mental health and public safety?" Show the RAG synthesis with direct quotes and citations.
  6. Teams Chatbot — Proofread — Paste the sample transcript. Show the corrected output with change explanations.

Key Talking Points

  • "This is a reference architecture." — Every Azure AI service in one platform: OpenAI (GPT-4o, embeddings, Whisper, Bing grounding), AI Search, Cosmos DB, APIM, Copilot Studio.
  • "Serverless and consumption-based." — No always-on VMs. Functions scale to zero. Cosmos DB is serverless. Cost tracks with usage.
  • "Secure by design." — Managed identity everywhere, VNet + private endpoints, APIM as the gateway, no secrets in code.
  • "Real data, real outlets." — 78 clips from 29 sources, not mock data. The web search runs against Bing's live index every morning.
  • "Built in days, not months." — IaC in Bicep, TypeScript backend, React dashboard. The whole platform deploys from a single repo.
  • "Extensible." — Each capability is a separate Function. Add a new data source? New Function, same pattern. Another office wants this? `az deployment group create` with different parameters.

If They Ask...

"Where do the clips come from?"
Two sources: governor.nc.gov press releases (scraped directly) + web search via Azure OpenAI Responses API with Bing grounding (20+ outlets). 5 focused queries per run targeting budget, healthcare, Helene recovery, public safety, and general coverage.
"Is the AI hallucinating?"
No. Clips are real articles with real URLs. Remarks search only quotes text that exists in uploaded documents. Proofread shows every change with a reason.
"What about Bing Search v7?"
Retired by Microsoft. We use the replacement: Azure OpenAI Responses API with built-in Bing grounding. Same Bing index, delivered through the AI model. No separate resource needed.
"How much does it cost?"
~$125-200/month total. The only fixed cost is AI Search at ~$70/mo. Everything else scales to zero. Web search adds ~$5/mo. Less than a single software license.
"Can we add more outlets?"
The web search already covers the entire internet — it's not limited to specific sites. As outlets publish articles about Governor Stein, Bing finds them. The 5 topic queries maximize coverage.
"What about the dashboard?"
React SPA with Tailwind CSS, proxied through APIM. Four tabs: overview stats, clips feed with filters, remarks documents, and ingestion run history. Auto-refreshes every 60 seconds.
