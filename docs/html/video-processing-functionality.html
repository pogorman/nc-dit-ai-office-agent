Video/Audio Transcription

Converting audio and video recordings to text using Azure OpenAI Whisper

What We Built

The Communications Office can now upload audio or video files and get an accurate text transcript back in seconds — no manual transcription needed.

This adds a fourth AI capability to the NC Comms Agent platform, alongside news clips monitoring, remarks search, and transcript proofreading. It uses OpenAI's Whisper model deployed on the existing Azure OpenAI resource — same managed identity auth, no new Azure resources or API keys.

25 MB file size limit — this is a Whisper API constraint. For longer recordings, the audio needs to be split into chunks before upload. A typical 25 MB MP3 file contains ~25–30 minutes of audio.
Language hint: Optionally include a language field (ISO 639-1 code like "en" or "es") to improve accuracy when the source language is known.

API Usage

How to call the transcription endpoint

Endpoint: POST https://nc-comms-agent-dev-apim.azure-api.net/comms/transcribe

Content-Type: multipart/form-data

Auth: Ocp-Apim-Subscription-Key header

Request (curl example)

curl -X POST \ "https://nc-comms-agent-dev-apim.azure-api.net/comms/transcribe" \ -H "Ocp-Apim-Subscription-Key: YOUR_KEY" \ -F "file=@press-conference.mp4" \ -F "language=en"

Response

{ "transcript": "Good afternoon, everyone. Today I am pleased to announce...", "filename": "press-conference.mp4", "fileSizeBytes": 15728640 }

Copilot Studio Integration

How to connect this from the Teams agent

The transcription endpoint uses multipart/form-data for file upload, which requires a different connector approach than the existing JSON-based actions (QueryClips, QueryRemarks, ProofreadTranscript).

Option A: Direct HTTP Action (Recommended)

  1. In Copilot Studio, create a new Topic triggered by phrases like "transcribe this recording", "convert video to text", "transcribe audio"
  2. Add an "Ask a question" node to collect the file upload from the user
  3. Add an HTTP request action (Power Automate cloud flow):
    • Method: POST
    • URI: https://nc-comms-agent-dev-apim.azure-api.net/comms/transcribe
    • Headers: Ocp-Apim-Subscription-Key
    • Body: multipart/form-data with the uploaded file
  4. Parse the JSON response and display the transcript field back to the user

Option B: Power Automate Flow (For Larger Files)

  1. Create a Power Automate cloud flow triggered from Copilot Studio
  2. User uploads file to a SharePoint library or OneDrive folder
  3. Flow detects the new file, reads it, and sends to the transcription endpoint
  4. Flow returns the transcript text back to Copilot Studio
  5. Advantage: bypasses Copilot Studio's message size limits for larger files
Why not add it to the existing custom connector? The current connector (apiDefinition.swagger.json) uses OpenAPI 2.0 with JSON request bodies. File uploads via multipart/form-data require different connector configuration. The existing 3 JSON-based actions (QueryClips, QueryRemarks, ProofreadTranscript) continue to work unchanged. Transcription can be added as a separate connector or handled via a Power Automate HTTP action.

Chain: Transcribe + Proofread

Two-step pipeline for polished transcripts

Whisper produces raw transcripts that may contain minor errors (especially for proper nouns, NC-specific terms, or cross-talk). The existing Proofread function can clean these up automatically.

→

→

proper nouns, punctuation

This chain can be automated in a Power Automate flow or built as a combined endpoint. The two functions already exist — they just need to be wired together.
