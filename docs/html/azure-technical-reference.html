1. Executive Summary

The NC Governor's Communications Agent is a serverless AI platform built for the NC Governor's Communications Office. It automates news monitoring, provides semantic search over historical speeches, proofreads transcripts, transcribes audio/video, and exposes an operational dashboard — all delivered through Microsoft Copilot Studio (in Teams and web) and a React SPA.

The Azure footprint is consumption-first and zero-trust:

  • 8 Azure Functions on Flex Consumption (Node.js 20, TypeScript strict mode)
  • Azure OpenAI with 4 model deployments (GPT-5-chat, GPT-4o, text-embedding-3-large, Whisper)
  • Azure AI Search (Basic tier) for hybrid vector + keyword search across 2 indexes
  • Cosmos DB (Serverless, NoSQL) with 4 containers
  • API Management (Consumption tier) as the single auth boundary — no Function endpoints exposed directly
  • VNet + Private Endpoints for Blob Storage and Cosmos DB
  • Managed Identity everywhere — no API keys, connection strings, or secrets in application code

Estimated steady-state cost: $125–200/month. All infrastructure is defined in Bicep under /infra.

4.1 clips-ingest.ts — News Aggregation

src/functions/clips-ingest.ts · 416 lines · Most complex function

Triggers:

  • Timer: 0 0 7 * * * — daily at 7 AM ET, timeframe = "past week"
  • HTTP POST: /api/clips/refresh — manual trigger, timeframe = "past 6 months"

Two-source aggregation pipeline:

                                     │

               Fetch HTML    Extract Text    Generate Embedding
              (10s timeout)  (Readability)   (title + lede)
                     │               │               │

                                     │
                          ┌──────────┴──────────┐
                          │                     │

Key functions:

Web search cost: ~$0.175/day (5 queries × $0.035/call) = ~$5/month.

4.2 clips-query.ts — Clip Search

src/functions/clips-query.ts · 114 lines

Trigger: HTTP POST /api/clips/query

Request body: { query, dateFrom?, dateTo?, top? }

Two modes:

  • Latest mode (query is empty, "latest", or "today"): Wildcard search on AI Search ordered by publishedAt DESC
  • Search mode: Generates embedding of query → hybrid search (vector + BM25) on the clips index with optional OData date filters

top is clamped to a maximum of 50. Returns { clips: NewsClip[], count }.

4.3 clips-digest.ts — Daily Digest (Stub)

src/functions/clips-digest.ts · 115 lines

Trigger: Timer 0 0 8 * * 1-5 (8 AM weekdays)

Queries Cosmos for clips published in the last 24 hours, generates an HTML table with outlet / title / lede / date columns. Email sending is not yet wired — the HTML is logged but not delivered. Intended integration: POST to a Logic App HTTP trigger that forwards via Outlook connector.

4.4 remarks-ingest.ts — Document Ingestion

src/functions/remarks-ingest.ts · 207 lines

Trigger: Blob Storage (remarks-uploads/{name})

Pipeline: Parse filename metadata → extract text → chunk by paragraph (200–1000 chars) → generate embedding per chunk → upsert to remarks-metadata + remarks-chunks containers.

Filename convention: YYYY-MM-DD_event-name_venue.txt — parsed via regex for automatic date, event, and venue extraction.

Known issue: Blob trigger is unreliable on Flex Consumption. Workaround: use seed/load-remarks.ts to manually process and index remarks files.

4.5 remarks-query.ts — Remarks RAG Search

src/functions/remarks-query.ts · 138 lines

Trigger: HTTP POST /api/remarks/query

Request body: { query, event?, dateFrom?, dateTo?, top?, synthesize? }

This is the core RAG (Retrieval-Augmented Generation) function:

  1. Embed the user's query
  2. Hybrid search on the remarks index (vector + BM25, optional OData filters)
  3. If synthesize=true (default): format top-K chunks with metadata, pass to GPT-5-chat with a system prompt instructing it to synthesize with direct quotes, citations (date + event), and messaging evolution tracking

Returns { synthesis?, sources: RemarksChunk[], count }. Synthesis temperature: 0.3 (conservative). Max tokens: 8192.

4.6 proofread.ts — Transcript Proofreading

src/functions/proofread.ts · 193 lines

Trigger: HTTP POST /api/proofread

Request body: { transcript, speakerLabels? }

Sends transcript to GPT-5-chat with a 6-rule (or 7-rule with speaker labels) system prompt targeting ASR/OCR errors: homophones, capitalization, punctuation, number formatting, garbled text. Temperature: 0.2 (very conservative). Max input: 100,000 characters.

Returns structured JSON: { corrected, changes: [{ original, corrected, confidence, reason }], summary }. No data is stored — purely stateless transformation.

4.7 transcribe.ts — Whisper Transcription

src/functions/transcribe.ts · 106 lines

Trigger: HTTP POST /api/transcribe (multipart/form-data)

Accepts: file field (mp3, mp4, wav, webm, m4a, mpeg, mpga) + optional language (ISO 639-1). Max file size: 25 MB (Whisper API limit).

Converts uploaded file to buffer, calls transcribeAudio() helper, returns { transcript, filename, fileSizeBytes }. No transcript is persisted — stateless.

4.8 dashboard.ts — Dashboard API

src/functions/dashboard.ts · 177 lines

Four read-only GET endpoints feeding the React dashboard. All query Cosmos DB directly (no AI Search).

5. Shared Modules & Singletons

Three client singletons in src/shared/ provide all Azure SDK interactions. Every function imports these rather than creating its own clients.

5.1 openai-client.ts

src/shared/openai-client.ts

Singleton AzureOpenAI client with lazy initialization. Auth: DefaultAzureCredential → getBearerTokenProvider with scope cognitiveservices.azure.com/.default.

ExportSignatureWhat It Does
getChatCompletion (systemPrompt, userPrompt, options?) → Promise<string> Chat completions via GPT4O_DEPLOYMENT_NAME. Auto-detects reasoning models (gpt-5*, o* prefixes): uses max_completion_tokens instead of max_tokens, drops temperature, defaults to 16384 tokens.
getEmbedding (text) → Promise<number[]> 3072-dimension vector via EMBEDDING_DEPLOYMENT_NAME (text-embedding-3-large)
webSearch (query) → Promise<WebSearchResult[]> Bing-grounded web search via Azure OpenAI Responses API. Uses OpenAI class (not AzureOpenAI) with baseURL: ${endpoint}/openai/v1/. Extracts URL citations from response annotations. search_context_size: "high".
transcribeAudio (buffer, filename, language?) → Promise<string> Whisper via WHISPER_DEPLOYMENT_NAME. Converts Buffer to File via toFile().
Responses API pattern: The webSearch() function creates a separate OpenAI client (not the singleton AzureOpenAI) because the Responses API requires the /openai/v1/ base URL path. The managed identity token is passed as the apiKey parameter.

5.2 search-client.ts

src/shared/search-client.ts

Per-index SearchClient<T> cache via Map. Auth: DefaultAzureCredential.

ExportSignatureWhat It Does
getSearchClient<T> (indexName) → SearchClient<T> Factory with caching. Returns cached client or creates new one for the given index.
hybridSearch<T> (indexName, query, vector, options?) → Promise<T[]> Combined BM25 keyword + vector (kNN on embedding field) search. Options: { filter?, top? }. Default top: 5.

5.3 cosmos-client.ts

src/shared/cosmos-client.ts

Three-level singleton: CosmosClient → Database → Container cache (Map). Auth: DefaultAzureCredential via aadCredentials parameter.

ExportSignatureWhat It Does
getContainer (containerName) → Container Only public export. Returns cached Container reference for the given name.

Database name defaults to comms-agent (configurable via COSMOS_DB_DATABASE env var). Container references are cached to avoid repeated lookups.

The system follows a zero-secrets-in-code model. Every Azure service interaction uses managed identity.

Orchestration

Copilot Studio uses generative orchestration — no manual topic routing. The OpenAPI descriptions in the connector drive tool selection. The orchestrator (GPT-4.0 in GCC) decides which connector action to invoke based on user intent.

Appendix D: Deployment Checklist

  1. Bicep deploy: az deployment group create -g {rg} -f infra/main.bicep -p infra/main.bicepparam
  2. Wait for RBAC propagation (~2–5 minutes for Cosmos sqlRoleAssignment to take effect)
  3. Create AI Search indexes: npx tsx seed/create-search-indexes.ts
  4. Flip Storage to public (temporarily, for local seeding): az storage account update -n nccommsstordev -g {rg} --public-network-access Enabled
  5. Seed data:
    • npx tsx seed/load-clips.ts
    • npx tsx seed/index-clips-to-search.ts
    • npx tsx seed/load-remarks.ts seed/remarks/{file}.txt (repeat per file)
  6. Deploy Function App: npm ci --omit=dev && npm run build && func azure functionapp publish nc-comms-agent-func-dev
    Kill zombie func.exe if archive hangs.
  7. Flip Storage back to private: az storage account update -n nccommsstordev -g {rg} --public-network-access Disabled
  8. Retrieve Function host key and store in Key Vault as function-host-key
  9. Update APIM named value to reference Key Vault secret
  10. Update Copilot Studio connector: pac connector update --api-definition connector/apiDefinition.swagger.json --api-properties connector/apiProperties.json
