Key Findings

1. GPT-5-chat produces richer synthesis than GPT-4o. The upgrade is most visible in remarks synthesis: more structured sections, markdown tables for evolution tracking, deeper contextual analysis of individual quotes. For clips queries, the difference is negligible since retrieval does not depend on the chat model. The GPT-5-chat backend is a clear upgrade for the highest-value use case (remarks analysis).
2. Generative orchestration preserves content quality well. Direct quotes, citations, thematic structure, and evolution tracking all survive the GPT-4.0 "last mile." The orchestrator adds value by blending clips + remarks into unified answers (Q1, Q4, Q5). Scored 4/4 on all clips and remarks questions. Only gap: the proofread change list gets flattened to a single summary (3/4 on Q3). For remarks synthesis — the highest-value use case — quality degradation is minimal.
3. Classic orchestration has higher fidelity when it works, but reliability is a problem. The synthesis text comes through with full quotes and numbered citations (Q7, Q2 were both excellent). Q2 required rephrasing to match the topic trigger, exposing a sensitivity to prompt wording. Classic gives you control over what fields to display but requires careful trigger phrase engineering. Once routed correctly, content quality is comparable to the direct API output.
4. The "last mile" problem is smaller than expected. The concern was that Copilot Studio's GPT-4.0 orchestrator would degrade the backend output. For remarks synthesis (the highest-value use case), the quality degradation is minimal. Quotes are preserved, citations are maintained, thematic structure survives. The GPT-5-chat backend does the heavy analytical lifting; Copilot Studio presents it faithfully.
